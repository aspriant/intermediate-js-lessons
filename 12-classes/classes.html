Javascript Classes

Back to Class

  • JS Classes are a syntactic improvement of prototypical inheritance
  • Composed of attributes and methods
  • Instances of a class are instantiated with the new keyword
    • const dog = new Dog()
  • Names are capitalized by convention

Static

  • Static methods are bound to the Class
  • They are not directly called by instantiated objects
    • Can be accessed via the constructor attribute
  • Generally serve a utlity purpose
    • Array.isArray()

Getters and Setter

  • Getters allow computed values to be accessed like attributes
  • Setters allow for custom functionality to be applied when setting an attribute
  • Utilized by adding a keyword before a function, who's name will be treated as an attribute
    • get
    • set

Inheritance

  • Classes may inherit functionality
  • A child will Extend a parent
  • The parent's constructor may be called via super(args)
    • From the child's constructor
  • The child may then access all methods and attributes from the parent

Private (Experimental)

  • Private attributes are an experimental feature
  • Begin the name with #
  • Private != Hidden
  • Should be free to use with transpiler like babel

Final Thoughts

  • Classes are often viewed as merely an inheritance tool for JS
  • They provide much more than that
  • When used properly, classes can help keep organized
